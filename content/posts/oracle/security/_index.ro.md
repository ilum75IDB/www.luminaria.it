În Oracle securitatea nu este o opțiune.<br>
Este o structură portantă a bazei de date, prezentă din prima zi.<br>

Modelul de securitate Oracle este probabil cel mai complet dintre bazele de date relaționale: utilizatori separați de scheme (dar legați), roluri predefinite și personalizate, privilegii de sistem și de obiect, profiluri de resurse, audit unificat.<br>

Puternic? Enorm.<br>
Configurat greșit în 90% din instalările pe care le-am văzut? De asemenea.<br>

Problema nu este lipsa instrumentelor. Este excesul. Oracle oferă atâtea opțiuni de securitate încât tentația este să simplifici cu un `GRANT DBA` și să treci la problema următoare. Am văzut asta făcut de dezvoltatori, administratori de sistem, chiar și de DBA-uri cu ani de experiență.<br>

În această secțiune povestesc cum se proiectează securitatea Oracle în medii reale: separarea rolurilor, principiul privilegiului minim, auditul și capcanele în care cad chiar și profesioniștii experimentați.<br>

Pentru că în Oracle securitatea nu se adaugă după.<br>
Se proiectează înainte. Sau se plătește după.
